Installing Gregorio under Linux

This page describes the installation and compilation of Gregorio software under a GNU/Linux system. Whether you install a precompiled version or whether you compile from source, it's a quick and easy operation.

Getting the sources

To obtain the sources of the stable version, you just need to download them from the download page, and decompress the archive.

To get the development version, create a local copy of the repository of the project, with the command:

git clone --depth 1 -b develop https://github.com/gregorio-project/gregorio.git gregorio-git
cd gregorio-git

Install dependencies

Under Ubuntu/Debian/Mint:

sudo aptitude install autotools-dev flex bison libltdl7-dev texlive-luatex autopoint python-fontforge fontforge autoconf
For gregorio versions <4.0, is you are using --enable-xml-read then this additional dependency is needed:
libxml2-dev
If you're using another distribution, some packages may have a slightly different name. I.e. under Ubuntu you can use apt-get instead of aptitude. If you're going to build gregorio from source code, you may need in addition following packages: fontforge cmake autoconf

Building

In the directory of the sources, run:

$ ./build.sh
$ sudo ./install.sh

Updating

To update your git version, you just need to go into the directory containing the source files (assuming you kept them), run git pull and recompile with the same commands.

Next steps
